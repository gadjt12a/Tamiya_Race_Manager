FIXED
  - Podium modal not appearing after Final race confirmed.
    Caused by a nested template literal with embedded quotes
    crashing showPodiumModal() silently before the modal opened.
    Points label logic extracted to a safe variable before render.
  - Added try/catch around computeFinalStandings() in podium
    so a standings error can no longer block the modal.

ADDED
  - 🏆 Show Results recovery button in race coordinator top bar.
    Appears whenever podium data is set. Allows coordinator to
    manually re-trigger the podium modal if it fails to appear.
  - 🏁 favicon added to browser tab — removes 404 console error.

ADDED
  - Double Points toggle (×2 PTS) on setup screen.
    Applies to all classes on that night.
    Top 10 points doubled (20→2 instead of 10→1).
    EVO Junior points also double when EVO is doubled.
    Shows ✕2 DOUBLE POINTS on podium screen.
    Shows ×2 badge on home screen event history.
  - Enriched JSON event structure for manual audit:
    · raceBreakdown: every individual race in
      chronological order with bracket and places.
    · racerSummary: full per-racer detail with points.
    · pointsScheme and doublePoints flags recorded.
    · Inline field comments for manual editing guidance.
  - Bundled Python for Windows — no installation needed.
    python-bundle folder included, app runs fully offline.
  - BUILD PYTHON BUNDLE (admin use only).bat — rebuilds
    the python-bundle if needed (requires internet once).
  - Mac launcher updated with clearer Python install guidance.
  - README.txt updated for both Windows and Mac.

FIXED
  - Special Event "Open (All Classes)" option removed —
    was confusing and non-functional.
  - Special events can now run without an active season,
    saved to a standalone specialEvents bucket in the DB.
  - Multiple classes under same special event name now
    group together correctly in the standings view.
  - Special event points scheme (Standard/Double/Podium)
    flows correctly through to exports.

 v9.12  (2026-04-19)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FIXED
  - 2nd Chance bracket logic: when mainPool===1 and
    secondPool===3, a single 3-person race was producing
    only 1 finalist instead of 2.
    Fix: racer with most wins receives a bye to the Final,
    the other 2 race for the second Final spot.
  - Verified with 1000 simulations across 5–21 racers.
    All finals correctly assembled as 1+2=3.

ADDED
  - Season points accumulation now aggregates by racer name
    (normalised) instead of temporary race-night ID.
    Racers now correctly accumulate points across events.
  - Export results — three formats from podium screen
    and season standings screen:
    · 💾 HTML — self-contained webpage, good for emailing.
    · 📊 CSV — spreadsheet, opens in Excel.
    · 🖨 Print/PDF — print-ready page with Print button.
  - Junior toggle now shows on all classes (not just EVO).
    Junior flag is a person attribute — affects points
    and standings in EVO class only.
  - Manage Roster modal — edit names, toggle Junior/Adult,
    bulk All Junior / All Adult with search filter, delete.
  - Junior toggle in roster sidebar for quick inline editing.
  - Auto-shutdown when browser tab is closed (heartbeat).
  - Fully offline operation — no internet required.
  - Mac launcher (Start Race Manager.command).
  - Fonts switched to system fonts (Segoe UI / Arial Narrow)
    — no internet required for font loading.
  - CHANGELOG.txt added (this file).
